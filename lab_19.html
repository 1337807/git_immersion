Lab 19: Amending Commits

Goals

  • Learn how to amend an existing commit

Steps

Change the program then commit

Add an author commit to the program.

File: hello.rb

# This is an unwanted but committed change
# Author: Jim Weirich
name = ARGV.first || "World"

puts "Hello, #{name}!"

Execute:

git add .
git commit -m 'Add an author comment'

Oops, Should have an Email

After you make the commit, you realize that any good author comment should have an email included. Update the hello program to include an email.

File: hello.rb

# This is an unwanted but committed change
# Author: Jim Weirich (jim@somewhere.com)
name = ARGV.first || "World"

puts "Hello, #{name}!"

Ammend the Previous Commit

We really don’t want a separate commit for just the email. Let’s amend the previous commit to include the email change.

Execute:

git add .
git commit --amend -m 'Add an author/email comment'

Output:

$ git add .
$ git commit --amend -m 'Add an author/email comment'
[master 40014be] Add an author/email comment
 1 files changed, 2 insertions(+), 1 deletions(-)

Review the History

Execute:

git hist

Output:

$ git hist
* 40014be - 2010-06-21 Add an author/email comment (HEAD, master) [Jim Weirich]
* 28cec91 - 2010-06-21 Added a comment (v1) [Jim Weirich]
* f91f4af - 2010-06-21 Added a default value (v1-beta) [Jim Weirich]
* e6d69ca - 2010-06-21 Using ARGV [Jim Weirich]
* 31b3e1b - 2010-06-21 First Commit [Jim Weirich]

We can see the original “author” commit is now gone, and it is replaced by the “author/email” commit. You can achieve the same effect by reseting the branch back one commit and then recommitting the new changes.
