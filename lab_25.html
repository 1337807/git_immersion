Lab 25: Navigating Branches

Goals

  • Learn how to navigate between the branches of a repository

Steps

You now have two branches in your project:

Execute:

git hist --all

Output:

$ git hist --all
* 164696a 2010-12-28 | Updated Rakefile (HEAD, greet) [Jim Weirich]
* 8dda11d 2010-12-28 | Hello uses Greeter [Jim Weirich]
* 87c19c4 2010-12-28 | Added greeter class [Jim Weirich]
* 19e2359 2010-12-28 | Added a Rakefile. (master) [Jim Weirich]
* 0530609 2010-12-28 | Moved hello.rb to lib [Jim Weirich]
* 2fc367b 2010-12-28 | Add an author/email comment [Jim Weirich]
* 15fde51 2010-12-28 | Added a comment (v1) [Jim Weirich]
* e76e760 2010-12-28 | Added a default value (v1-beta) [Jim Weirich]
* 2370a1d 2010-12-28 | Using ARGV [Jim Weirich]
* a3f8b17 2010-12-28 | First Commit [Jim Weirich]

Switch to the Master Branch

Just use the git checkout command to switch between branches.

Execute:

git checkout master
cat lib/hello.rb

Output:

$ git checkout master
Switched to branch 'master'
$ cat lib/hello.rb
# Default is World
# Author: Jim Weirich (jim@somewhere.com)
name = ARGV.first || "World"

puts "Hello, #{name}!"

You are now on the master branch. You can tell because the hello.rb file doesn’t use the Greeter class.

Switch Back to the Greet Branch.

Execute:

git checkout greet
cat lib/hello.rb

Output:

$ git checkout greet
Switched to branch 'greet'
$ cat lib/hello.rb
require 'greeter'

# Default is World
name = ARGV.first || "World"

greeter = Greeter.new(name)
puts greeter.greet

The contents of the lib/hello.rb confirms we are back on the greet branch.
