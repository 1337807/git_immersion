Lab 32: Resetting the Greet Branch

Goals

  • Reset the greet branch to the point before the first merge.

Steps

Reset the greet branch

Let’s go back in time on the greet branch to the point before we merged master onto it. We can reset a branch to any commit we want. Essentially this is modifying the branch pointer to point to anywhere in the commit tree.

In this case we want to back greet up to the point prior to the merge with master. We need to find the last commit before the merge.

Execute:

git hist

Output:

$ git hist
*   27fb064 2010-06-21 | Merged master fixed conflict. (HEAD, greet) [Jim Weirich]
|\  
| * 8e354bc 2010-06-21 | Made interactive (master) [Jim Weirich]
* |   6fde7c0 2010-06-21 | Merge branch 'master' into greet [Jim Weirich]
|\ \  
| |/  
| * f5019e3 2010-06-21 | Added README [Jim Weirich]
* | 6fe71ad 2010-06-21 | Updated Rakefile [Jim Weirich]
* | 086d24e 2010-06-21 | Hello uses Greeter [Jim Weirich]
* | ef49107 2010-06-21 | Added greeter class [Jim Weirich]
|/  
* a11330b 2010-06-21 | Added a Rakefile. [Jim Weirich]
* a9bf23c 2010-06-21 | Moved hello.rb to lib [Jim Weirich]
* a9cd2bf 2010-06-21 | Add an author/email comment [Jim Weirich]
* 4fbdc1a 2010-06-21 | Added a comment (v1) [Jim Weirich]
* 5e3a072 2010-06-21 | Added a default value (v1-beta) [Jim Weirich]
* 09616c1 2010-06-21 | Using ARGV [Jim Weirich]
* 755c045 2010-06-21 | First Commit [Jim Weirich]

That’s a bit hard to read, but looking at the data we see that the “Updated Rakefile” commit was the last commit on the greet branch before merging. Let’s reset the greet branch to that commit.

Execute:

git reset --hard <hash>

Output:

$ git reset --hard 6fe71ad
HEAD is now at 6fe71ad Updated Rakefile

Check the branch.

Look at the log for the greet branch. We no longer have the merge commits in its history.

Execute:

git hist --all

Output:

$ git hist --all
* 8e354bc 2010-06-21 | Made interactive (master) [Jim Weirich]
