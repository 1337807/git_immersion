Lab 32: Resetting the Greet Branch

Goals

  • Reset the greet branch to the point before the first merge.

Steps

Reset the greet branch

Let’s go back in time on the greet branch to the point before we merged master onto it. We can reset a branch to any commit we want. Essentially this is modifying the branch pointer to point to anywhere in the commit tree.

In this case we want to back greet up to the point prior to the merge with master. We need to find the last commit before the merge.

Execute:

git hist

Output:

$ git hist
*   cf2b29c 2010-06-21 | Merged master fixed conflict. (HEAD, greet) [Jim Weirich]
|\  
| * 936c45f 2010-06-21 | Made interactive (master) [Jim Weirich]
* |   247fc6f 2010-06-21 | Merge branch 'master' into greet [Jim Weirich]
|\ \  
| |/  
| * f396c14 2010-06-21 | Added README [Jim Weirich]
* | 1b2b22d 2010-06-21 | Updated Rakefile [Jim Weirich]
* | 8cc8712 2010-06-21 | Hello uses Greeter [Jim Weirich]
* | 5563881 2010-06-21 | Added greeter class [Jim Weirich]
|/  
* 8547b5f 2010-06-21 | Added a Rakefile. [Jim Weirich]
* c2ad33c 2010-06-21 | Moved hello.rb to lib [Jim Weirich]
* 2ae006e 2010-06-21 | Add an author/email comment [Jim Weirich]
* 7c5f85f 2010-06-21 | Added a comment (v1) [Jim Weirich]
* 7d21264 2010-06-21 | Added a default value (v1-beta) [Jim Weirich]
* 46d1c17 2010-06-21 | Using ARGV [Jim Weirich]
* c97dcf3 2010-06-21 | First Commit [Jim Weirich]

That’s a bit hard to read, but looking at the data we see that the “Updated Rakefile” commit was the last commit on the greet branch before merging. Let’s reset the greet branch to that commit.

Execute:

git reset --hard <hash>

Output:

$ git reset --hard 1b2b22d
HEAD is now at 1b2b22d Updated Rakefile

Check the branch.

Look at the log for the greet branch. We no longer have the merge commits in its history.

Execute:

git hist --all

Output:
