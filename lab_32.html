Lab 32: Resetting the Greet Branch

Goals

  • Reset the greet branch to the point before the first merge.

Steps

Reset the greet branch

Let’s go back in time on the greet branch to the point before we merged master onto it. We can reset a branch to any commit we want. Essentially this is modifying the branch pointer to point to anywhere in the commit tree.

In this case we want to back greet up to the point prior to the merge with master. We need to find the last commit before the merge.

Execute:

git checkout greet
git hist

Output:

$ git checkout greet
Already on 'greet'
$ git hist
*   a042980 2010-12-28 | Merged master fixed conflict. (HEAD, greet) [Jim Weirich]
|\  
| * a0d9fc3 2010-12-28 | Made interactive (master) [Jim Weirich]
* |   5602bd6 2010-12-28 | Merge branch 'master' into greet [Jim Weirich]
|\ \  
| |/  
| * 399b53b 2010-12-28 | Added README [Jim Weirich]
* | 164696a 2010-12-28 | Updated Rakefile [Jim Weirich]
* | 8dda11d 2010-12-28 | Hello uses Greeter [Jim Weirich]
* | 87c19c4 2010-12-28 | Added greeter class [Jim Weirich]
|/  
* 19e2359 2010-12-28 | Added a Rakefile. [Jim Weirich]
* 0530609 2010-12-28 | Moved hello.rb to lib [Jim Weirich]
* 2fc367b 2010-12-28 | Add an author/email comment [Jim Weirich]
* 15fde51 2010-12-28 | Added a comment (v1) [Jim Weirich]
* e76e760 2010-12-28 | Added a default value (v1-beta) [Jim Weirich]
* 2370a1d 2010-12-28 | Using ARGV [Jim Weirich]
* a3f8b17 2010-12-28 | First Commit [Jim Weirich]

That’s a bit hard to read, but looking at the data we see that the “Updated Rakefile” commit was the last commit on the greet branch before merging. Let’s reset the greet branch to that commit.

Execute:

git reset --hard <hash>

Output:

$ git reset --hard 164696a
HEAD is now at 164696a Updated Rakefile

Check the branch.

Look at the log for the greet branch. We no longer have the merge commits in its history.

Execute:

git hist --all

Output:
