Lab 32: Reseting the Greet Branch

Goals

  • Reset the greet branch to the point before the first merge.

Steps

Reset the greet branch

Let’s go back in time on the greet branch to the point before we merged master onto it. We can reset a branch to any commit we want. Essentially this is modifying the branch pointer to point to anywhere in the commit tree.

In this case we want to back greet up to the point prior to the merge with master. We need to find the last commit before the merge.

Execute:

git hist

Output:

$ git hist
*   cf33651 - 2010-06-20 Merged master fixed conflict. (HEAD, greet) [Jim Weirich]
|\  
| * 5456cbc - 2010-06-20 Made interactive (master) [Jim Weirich]
* |   c8f9eed - 2010-06-20 Merge branch 'master' into greet [Jim Weirich]
|\ \  
| |/  
| * 10f5258 - 2010-06-20 Added README [Jim Weirich]
* | 2e80a49 - 2010-06-20 Updated Rakefile [Jim Weirich]
* | 33bb025 - 2010-06-20 Hello uses Greeter [Jim Weirich]
* | 9d4bc89 - 2010-06-20 Added greeter class [Jim Weirich]
|/  
* c912f5d - 2010-06-20 Added a Rakefile. [Jim Weirich]
* 3af42b0 - 2010-06-20 Moved hello.rb to lib [Jim Weirich]
* 74cf292 - 2010-06-20 Add an author/email comment [Jim Weirich]
* 20f2972 - 2010-06-20 Added a comment (v1) [Jim Weirich]
* c69b1e9 - 2010-06-20 Added a default value (v1-beta) [Jim Weirich]
* 9e65ccd - 2010-06-20 Using ARGV [Jim Weirich]
* 5fb8ac7 - 2010-06-20 First Commit [Jim Weirich]

That’s a bit hard to read, but looking at the data we see that the “Updated Rakefile” commit was the last commit on the greet branch before merging. Let’s reset the greet branch to that commit.

Execute:

git reset --hard <hash>

Output:

$ git reset --hard 2e80a49
HEAD is now at 2e80a49 Updated Rakefile

Check the branch.

Look at the log for the greet branch. We no longer have the merge commits in its history.

Execute:

git hist --all

Output:

$ git hist --all
* 5456cbc - 2010-06-20 Made interactive (master) [Jim Weirich]
