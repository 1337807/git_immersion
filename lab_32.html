Lab 32: Resetting the Greet Branch

Goals

  • Reset the greet branch to the point before the first merge.

Steps

Reset the greet branch

Let’s go back in time on the greet branch to the point before we merged master onto it. We can reset a branch to any commit we want. Essentially this is modifying the branch pointer to point to anywhere in the commit tree.

In this case we want to back greet up to the point prior to the merge with master. We need to find the last commit before the merge.

Execute:

git checkout greet
git hist

Output:

$ git checkout greet
Already on 'greet'
$ git hist
*   84559c5 2010-12-28 | Merged master fixed conflict. (HEAD, greet) [Jim Weirich]
|\  
| * 339d25d 2010-12-28 | Made interactive (master) [Jim Weirich]
* |   3a5090d 2010-12-28 | Merge branch 'master' into greet [Jim Weirich]
|\ \  
| |/  
| * cdc2714 2010-12-28 | Added README [Jim Weirich]
* | dad9364 2010-12-28 | Updated Rakefile [Jim Weirich]
* | c4f32a5 2010-12-28 | Hello uses Greeter [Jim Weirich]
* | 51d5c49 2010-12-28 | Added greeter class [Jim Weirich]
|/  
* d4258dd 2010-12-28 | Added a Rakefile. [Jim Weirich]
* 0179abf 2010-12-28 | Moved hello.rb to lib [Jim Weirich]
* f5505ca 2010-12-28 | Add an author/email comment [Jim Weirich]
* 2d930c3 2010-12-28 | Added a comment (v1) [Jim Weirich]
* 9ff05d6 2010-12-28 | Added a default value (v1-beta) [Jim Weirich]
* 7ce3917 2010-12-28 | Using ARGV [Jim Weirich]
* 8613c73 2010-12-28 | First Commit [Jim Weirich]

That’s a bit hard to read, but looking at the data we see that the “Updated Rakefile” commit was the last commit on the greet branch before merging. Let’s reset the greet branch to that commit.

Execute:

git reset --hard <hash>

Output:

$ git reset --hard dad9364
HEAD is now at dad9364 Updated Rakefile

Check the branch.

Look at the log for the greet branch. We no longer have the merge commits in its history.

Execute:

git hist --all

Output:

$ git hist --all
