Lab 35: Merging Back to Master

Goals

  • We’ve kept our greet branch up to date with master (via rebase), now lets merge the greet changes back into the master branch.

Steps

h3 Merge greet into master

Execute:

git checkout master
git merge greet

Output:

$ git checkout master
Switched to branch 'master'
$
$ git merge greet
Updating d0a15c2..47de439
Fast-forward
 Rakefile       |    2 +-
 lib/greeter.rb |    8 ++++++++
 lib/hello.rb   |    8 +++++---
 3 files changed, 14 insertions(+), 4 deletions(-)
 create mode 100644 lib/greeter.rb

Because the head of master is a direct ancestor of the head of the greet branch, git is able to do a fast forward merge. When fast-forwarding, the branch pointer is simply moved forward to point to same commit as the greeter branch.

There will never be conflicts in a fast forward merge.

git hist
